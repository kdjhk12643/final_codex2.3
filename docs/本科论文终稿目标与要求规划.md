# 本科论文终稿目标与要求规划

## 1. 最终目标

论文终稿应形成一份主题明确、结构完整、数据和代码结果一致、图表齐全、格式可提交的本科毕业论文。论文题目为《基于负荷特性的地铁车站通风空调系统容量配置优化研究》，研究对象限定为地铁车站站台区域通风空调系统，技术路线严格对应任务书要求：

数据准备 -> 影响因素分析 -> 负荷分解与特征构建 -> 负荷曲线聚类 -> LSTM 负荷预测 -> NSGA-II 容量优化 -> TOPSIS 方案排序 -> 工程评价。

终稿交付物包括：

1. 论文终稿 DOCX。
2. 与 DOCX 对应的中间 Markdown 稿。
3. 论文所用图表、实验数据表和模型输出。
4. 终稿生成脚本。
5. 本规划文档，作为后续检查和迭代依据。

## 3. 论文终稿结构要求

在未提供学校固定模板的情况下，按通用本科毕业论文结构整理：

1. 封面：题目、论文类型、学生姓名、专业班级、指导教师、完成日期。
2. 目录：保留目录页，最终可在 Word 中更新。
3. 中文摘要与关键词：概括研究对象、方法、结果和工程意义。
4. 第 1 章 绪论：研究背景、目的意义、研究现状、研究内容、技术路线。
5. 第 2 章 理论基础：系统组成、负荷构成、影响因素、聚类、预测和优化理论。
6. 第 3 章 数据预处理与负荷特性分析：数据来源、预处理、Pearson、负荷分解、K-Means。
7. 第 4 章 负荷预测模型：输入特征、LSTM、BP、指标和结果对比。
8. 第 5 章 容量配置优化模型：决策变量、目标函数、约束、NSGA-II、TOPSIS。
9. 第 6 章 优化结果与工程评价：方案对比、节能降本、设备匹配、改造适配性。
10. 第 7 章 结论与展望：主要结论、不足与后续研究。
11. 参考文献：来源需与正文研究现状和方法描述匹配。
12. 附录：建议放置主要代码文件、关键参数表或复现实验说明。

## 4. 内容质量要求

1. 题目、研究对象、算法链和结果指标必须全文一致。
2. 所有结果数字必须优先来自 `output/tables`，不得与代码输出冲突。
3. 不保留“占位”“待填”等终稿痕迹。
4. 图表必须有编号、标题和正文引用。
5. 关键公式必须说明变量含义。
6. 结果分析不能只罗列数字，必须解释工程意义。
7. 对数据范围、能耗估算简化、工程现场校核不足等限制应在第 7 章如实说明。

## 6. 执行步骤

### 步骤 1：结果一致性检查

检查任务书、代码、`output/tables` 和论文正文中的核心指标是否一致。重点核对 LSTM/BP 指标、聚类 K 值、基准/优化方案容量、成本、冗余率和能耗。

### 步骤 2：清除终稿占位

修改论文生成脚本，将已有 PNG 图插入 DOCX，将流程类占位替换为可视化示意图或正式文字说明，补齐设备配置明细表，删除“扩写稿”“占位”“待填”等非终稿表述。

### 步骤 3：生成论文终稿

运行终稿生成脚本，输出最终 DOCX 和中间 Markdown。生成文件名应明确标识为“本科论文终稿”。

### 步骤 4：渲染与格式检查

将 DOCX 渲染为 PDF/PNG 页面，检查是否存在文字溢出、表格拥挤、图片缺失、页眉页脚异常、乱码和明显空白。

### 步骤 5：最终审计

按任务书逐项核对代码、图表、章节和终稿交付物，确认无缺项后再视为完成。

## 7. 验收标准

终稿视为完成必须同时满足：

1. DOCX 能正常打开，文件名为本科论文终稿。
2. 全文无“占位”“待填”等未完成标记。
3. 任务书 7 项技术任务均在正文中有对应章节和结果。
4. 论文中的核心指标与 `output/tables` 一致。
5. 主要图表已插入或以正式表格呈现。
6. 生成脚本可复现终稿。
7. 已完成渲染检查；若渲染工具不可用，需说明未完成的验证风险。
